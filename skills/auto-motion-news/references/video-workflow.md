# Video Workflow

Use this reference when turning a script, card-news project, source URL, or notes into a short-form motion-video package.

## Target Defaults

Default to vertical short-form video:

- Format: 9:16
- Duration: 30-60 seconds
- Primary outputs: MP4, caption, source notes
- Secondary outputs: scene preview images, PNG fallbacks when useful

Ask before switching to horizontal YouTube, long-form explainers, or square formats.

## Input Types

### Script

Tighten the first 2 seconds first. A script that starts with background context usually needs a stronger hook.

Convert the script into scenes:

- Scene 1: stop-scroll hook
- Scene 2: "what is this?"
- Scenes 3-6: examples, proof, or useful steps
- Scene 7: channel interpretation
- Scene 8: CTA

### Card-News Project

Read the card-news project as a story spine. Do not read cards aloud line by line.

Map card roles into video beats:

- Hook card -> first 2 seconds
- Situation card -> viewer problem
- Example/proof cards -> visual body scenes
- Channel take card -> interpretation
- CTA card -> final action

### Raw Source

When the user provides a source URL or report, summarize only after defining the viewer frame. The video should be about why the viewer cares, not about reciting the source.

## Scene Plan Rules

Each scene needs:

- Timestamp range
- Spoken or subtitle text
- Visual idea
- Motion idea
- Source or asset
- Engine choice
- Viewer reason

Most scenes should be 2-6 seconds. If one scene needs more than 8 seconds, split it unless it is a deliberate demo clip.

## Script Style

Write like a person talking, not like a press release.

For AI쭌-style AI information:

- Easy and casual
- Tutor-like
- Practical
- Slightly cute when appropriate
- Clear about what viewers can try

Avoid:

- Long abstract sentences
- Tool jargon without immediate plain-language translation
- "AI is changing everything" filler
- Unverified hype
- Reading captions as narration

## Approval Flow

Show the script and scene plan first. Do not jump straight to final render.

After script approval:

1. Create or gather scene assets.
2. Create preview motion.
3. Verify readable text and scene rhythm.
4. Ask for motion approval.
5. Render final MP4.
